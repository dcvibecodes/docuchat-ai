# Deployment Guide — Moving to Another Computer or VPS

## What to Copy

Copy the entire `document-chatbot` folder **except** `node_modules/`:

```
document-chatbot/
├── data/chatbot.db          ← ALL data (users, embeddings, config, chat logs, API keys)
├── uploads/                 ← Original uploaded files (needed for re-embedding)
├── src/                     ← Server code
├── public/                  ← Frontend
├── package.json
├── package-lock.json
├── .env                     ← PORT and SESSION_SECRET
└── .env.example
```

**Do NOT copy:** `node_modules/` — it will be regenerated by `npm install`.

### What's inside `data/chatbot.db`

This single file contains everything:
- All user accounts and passwords (hashed)
- All document metadata, chunks, and embeddings
- All chat history and chat logs
- System configuration (API keys, model settings, system prompt)
- Everything configured via the Admin UI

If you lose this file, you lose everything. **Back it up.**

## Setup on Computer B / VPS

### Prerequisites

- Node.js 18+ installed
- Internet access (for OpenAI API calls during chat)

### Steps

```bash
# 1. Copy the folder to the new machine (rsync, scp, USB, whatever)
scp -r document-chatbot/ user@your-vps:/home/user/

# 2. SSH into the machine (if VPS)
ssh user@your-vps

# 3. Navigate to the folder
cd document-chatbot

# 4. Install dependencies
npm install

# 5. (Optional) Edit .env if you need a different port
#    nano .env
#    PORT=3000
#    SESSION_SECRET=your-random-secret-string

# 6. Start the server
node src/index.js
```

The app is now running at `http://localhost:3000` (or whatever port you set).

## Running Permanently (VPS)

### Option 1: PM2 (recommended)

```bash
# Install PM2 globally
npm install -g pm2

# Start the app
pm2 start src/index.js --name dochat

# Save the process list (survives reboot)
pm2 save

# Generate startup script (auto-start on boot)
pm2 startup
# Follow the instructions it prints

# Useful commands:
pm2 status          # Check if running
pm2 logs dochat     # View logs
pm2 restart dochat  # Restart after code changes
pm2 stop dochat     # Stop the app
```

### Option 2: systemd (Linux)

Create `/etc/systemd/system/dochat.service`:

```ini
[Unit]
Description=DocChat AI
After=network.target

[Service]
Type=simple
User=your-username
WorkingDirectory=/home/your-username/document-chatbot
ExecStart=/usr/bin/node src/index.js
Restart=on-failure
RestartSec=10
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

Then:
```bash
sudo systemctl daemon-reload
sudo systemctl enable dochat
sudo systemctl start dochat
sudo systemctl status dochat    # Check status
sudo journalctl -u dochat -f   # View logs
```

## Reverse Proxy with HTTPS (Production)

For production, put Nginx in front with SSL:

```nginx
server {
    listen 80;
    server_name chat.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name chat.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/chat.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/chat.yourdomain.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # Important for SSE streaming
        proxy_buffering off;
        proxy_read_timeout 300s;
    }

    # Allow large file uploads
    client_max_body_size 50M;
}
```

Get a free SSL certificate:
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d chat.yourdomain.com
```

## Environment Variables

Only two are required in `.env`:

| Variable | Purpose |
|----------|---------|
| `PORT` | Server port (default: 3000) |
| `SESSION_SECRET` | Random string for session encryption. Change this from the dev default. |

Generate a good secret:
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

All other settings (API keys, models, prompt, thresholds) are managed through the Admin UI — no env vars needed.

## Updating the Code

If you make code changes on your dev machine and want to deploy:

```bash
# On dev machine: copy changed files (or use git)
scp -r src/ public/ package.json user@your-vps:/home/user/document-chatbot/

# On VPS: restart
pm2 restart dochat
# or
sudo systemctl restart dochat
```

**Do NOT overwrite `data/chatbot.db`** unless you want to replace all data.

## Backup Strategy

```bash
# Simple backup script (run via cron)
#!/bin/bash
BACKUP_DIR=/home/user/backups
DATE=$(date +%Y%m%d_%H%M%S)
cp /home/user/document-chatbot/data/chatbot.db $BACKUP_DIR/chatbot_$DATE.db
tar -czf $BACKUP_DIR/uploads_$DATE.tar.gz /home/user/document-chatbot/uploads/

# Keep last 7 days
find $BACKUP_DIR -name "chatbot_*.db" -mtime +7 -delete
find $BACKUP_DIR -name "uploads_*.tar.gz" -mtime +7 -delete
```

Add to crontab (`crontab -e`):
```
0 2 * * * /home/user/backup-dochat.sh
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| Port already in use | Change PORT in .env, or kill the existing process |
| Permission denied on uploads/ | `chmod -R 755 uploads/` |
| Can't connect from browser | Check firewall: `sudo ufw allow 3000` (or your port) |
| Sessions lost on restart | Expected — users re-login. Data is preserved. |
| Embeddings not working on VPS | Check that the VPS can reach api.openai.com (no firewall/proxy blocking) |
| Large files fail to upload | Check Nginx `client_max_body_size` if using reverse proxy |
